# v0.1 Verification Record

## Release Boundary

This record covers the complete `v0.1-alpha` and `v0.1-beta` scope in
`docs/roadmap.md`. Android/iOS enablement, visual rendering integrations,
engine-specific adapters beyond the Wesnoth scenario overlay, binary document
formats, image/audio localization, MCP, and the community registry remain v0.2
or later work.

## Alpha Evidence

| Requirement | Evidence |
| --- | --- |
| Protocol schemas and examples | 11 schemas, 11 matching examples, zero-dependency `validate-protocol` command |
| Lightweight reference CLI | Installed `localize-anything` console entrypoint and module CLI |
| Four-layer memory | Preflight creates context, glossary, JSONL TM, and canonical delivery manifest |
| Preflight and batch contracts | Inventory, non-text assets, workflow recommendation, batch plan, and work-packet schemas/runtime |
| Stable IDs and source hashes | Format-specific stable segment IDs plus SHA-256 and incremental diff tests |
| Staged rebuild and QA | Adapter rebuild commands target explicit staging paths; deterministic QA blocks structural drift |
| Immutable package | Atomic fail-on-existing snapshots with content hashes and mandatory package assets |
| Incremental update detection | New, unchanged, changed, moved, and deleted classifications |
| Review import and sign-off | Reviewed TM promotion plus manifest-hash-bound scoped acceptance |
| Apply dry-run | Create/replace/unchanged/conflict plan with backup metadata and no project mutation |
| The South Guard benchmark | Pinned upstream commit, POT/reference hashes, blind source gate, E0 verifier |

`review_ready` packaging requires both deterministic runtime/adapter evidence and
agent linguistic QA evidence. Only a separate user-owned scoped acceptance
record produces `user_accepted`.

## Reproducible Gates

Run from the repository root:

```bash
python3 -m unittest discover -s tests -v
python3 -m runtime.localize_anything validate-protocol
python3 -m runtime.localize_anything validate-contracts
python3 -m compileall -q runtime benchmarks
python3 /path/to/skill-creator/scripts/quick_validate.py skills/localize-anything
```

The public benchmark additionally runs:

```bash
python3 benchmarks/wesnoth-south-guard/prepare.py source /tmp/wesnoth-v01
python3 benchmarks/wesnoth-south-guard/verify.py source /tmp/wesnoth-v01
```

The benchmark verifier must report the pinned commit, `blind_boundary: pass`,
zero unresolved WML-context warnings, and the expected source segment count.
