# v0.3.0 Planning: Real-Project Workflow Hardening

## Purpose

v0.3.0 should focus on proving and hardening Localize Anything on real projects,
not on expanding support for every format.

Core theme:

> v0.3.0 = real-project workflow hardening

This is a public, high-level planning note. It describes a proposed direction,
not a commitment that the work or release is complete.

## Milestone 1 — AntennaPod disposable-clone full smoke test

### Goal

Run the documented AntennaPod smoke-test workflow on a disposable clone and
record reproducible evidence without committing generated outputs.

### Deliverable

`docs/antennapod-smoke-test-results.md`

The results document should record:

- AntennaPod commit hash;
- Localize Anything commit hash;
- commands run;
- detected Android resource files;
- source sets and qualifiers detected;
- number of generated segments, if generation is run;
- number of `owner_review_required` segments;
- unsupported complex-markup findings;
- presence of the review sheet and risk metadata;
- whether source files were modified;
- where generated outputs were stored;
- validation gates run;
- known limitations observed.

Rules:

- Do not vendor AntennaPod.
- Do not commit generated delivery packages.
- Do not commit translated AntennaPod resources.
- Do not run destructive apply commands unless explicitly operating on a
  disposable copy.
- Prefer evidence summaries over large artifacts.

## Milestone 2 — Workflow friction audit

After the smoke test, document:

- confusing CLI steps;
- missing commands;
- unclear output paths;
- unsafe defaults;
- places where users might accidentally modify source files;
- places where review artifacts are hard to find.

## Milestone 3 — Minimal runtime hardening

Only after the friction audit, implement the smallest runtime fixes justified by
real-project evidence.

Potential examples, subject to evidence and separate review:

- clearer output-directory handling;
- stronger dry-run defaults;
- a better smoke-test summary artifact;
- improved review-sheet discoverability;
- safer apply confirmation.

## Non-goals for v0.3.0

- full HTML parser;
- arbitrary Android resource support;
- layout, drawable, or asset localization;
- iOS expansion unless separately scoped;
- translation quality scoring;
- enterprise TMS features.

v0.3.0 should be driven by real smoke-test evidence, not speculative feature
expansion.
